# 计划：查看与分析 EQ Skills 项目

本计划旨在对 `eq-skills` 项目进行基础结构与当前开发状态的深度分析，并明确下一步实现步骤。

## 当前项目概览

通过对项目文件的初步检索，得出项目如下核心信息：
1. **项目名称**: `eq-skills` (v1.0.0)
2. **核心定位**: 为 AI 代理（如 Claude Code / Kilo 等）提供扩展的技能与命令包。
3. **主要组件**:
   - `bin/cli.js`: 命令行入口。目前支持 `eq init`（将 skills 和 commands 复制到 `.claude/` 目录）、`eq list`（列出支持的 skills/commands）和 `eq help`。
   - `skills/analyze-frontend/SKILL.md`: 核心的前端深度分析指令，提供了长达 9 个章节的前端分析逻辑。
   - `commands/eq/analyze-frontend.md`: 与上述 skill 绑定的命令入口。
4. **TODO 任务待办** (`TODO.md`):
   - 全局非全局（可能指命令安装/初始化的范围：是全局安装、当前项目 `.claude/` 还是支持 `.kilo/` 等其他 AI 代理工具配置路径）
   - 多个 agents（支持不同类型的 AI 代理或子代理机制）
   - typescript（将 CLI 重构或扩展为 TypeScript，或者支持 TypeScript 的 skills/commands 编写与编译）
   - cli交互逻辑 选择等（完善命令行交互逻辑，如命令行参数解析、交互式选择等，例如使用 inquirer/prompts 等库）
