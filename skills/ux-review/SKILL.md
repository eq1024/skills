# UX 体验审查 (ux-review)

这是一项在功能开发完成后，对涉及的前端修改进行统一检查与优化的任务。**在不改变任何业务逻辑的前提下，只打磨和优化交互体验**。

审查的范围默认聚焦于**本次前端修改（如 Git 未提交的变更、当前对话涉及的组件）**，同时也非常支持用户在调用时显式补充或指定特定的检查范围。

从以下七个维度逐一排查并修复：

## 执行流程

1. **确定范围（智能识别）**：
   - **自动提取**：主动检查当前工作区，通过 Git 状态（如 `git status`、`git diff`）自动提取本次修改中涉及的前端页面、组件及样式文件。
   - **上下文分析**：分析当前对话历史，找出本次对话中创建、编辑或讨论过的前端文件。
   - **补充范围**：结合用户在调用该技能时显式补充或指定的其他需要检查的文件、组件或目录。
   - **兜底逻辑**：若未发现未提交的修改或对话上下文中的文件，则要求用户提供需要检查的页面、组件或入口文件。
2. **逐维度审查**：对确定范围内的所有文件，按七个维度逐一进行检查。每个维度先列出发现的问题，再逐一进行修复。
3. **输出报告**：每个维度以表格形式列出检查项、结果和修复状态。

---

## 维度 1：加载态

| 检查项 | 说明 |
|---|---|
| 全页阻塞式 loading | 是否存在全页 spinner 阻塞用户操作？能改为骨架屏或局部 spinner？ |
| 子组件独立加载状态 | 子组件/子模块是否有独立的加载状态，避免父组件 loading 时子组件一片空白？ |
| 加载→空态过渡 | 是否存在 loading 闪现后又因数据为空变成空态？加载态→空态的过渡是否平滑？ |
| 重复触发请求 | 是否有重复触发请求导致 loading 闪烁（如 onMounted + watch immediate 同时触发）？ |

---

## 维度 2：错误态

| 检查项 | 说明 |
|---|---|
| 静默白屏/空态 | API 请求失败后页面是否静默白屏或显示空态？应展示错误信息 + 重试按钮 |
| 错误信息可读性 | 优先展示后端返回的中文消息，fallback 到通用提示，避免展示原始错误码 |
| 局部错误隔离 | 局部模块（如列表中的某个卡片）请求失败不应拖垮整页 |

---

## 维度 3：空态

| 检查项 | 说明 |
|---|---|
| 合理占位展示 | 列表为空、字段缺失时是否有合理的占位（el-empty / "-"）？ |
| 空态文案友好 | 空态提示文案是否友好、有引导性（而非干巴巴的"暂无数据"）？ |

---

## 维度 4：边界与守卫

| 检查项 | 说明 |
|---|---|
| 可选链 `?.` | 数组/嵌套对象访问是否加了可选链保护？ |
| v-for/v-if 安全 | 依赖异步数据的 `v-for`/`v-if` 是否在数据未就绪时不会报错？ |
| props 空值守卫 | props 传入空值/undefined 时子组件是否会发无效请求或崩溃？需加 watch 守卫或条件判断 |
| 工具函数兜底 | 时间格式化、金额转换等工具函数是否对 null/undefined 做了兜底？ |

---

## 维度 5：响应式与渲染

| 检查项 | 说明 |
|---|---|
| props 解构响应性 | 是否存在解构 props 导致响应性丢失（Vue `toRefs` / React 直接解构）？ |
| 切换刷新 | 切换 tab / 切换筛选条件后数据是否正确刷新？ |
| 重复渲染 | key 设置是否合理？是否存在不必要的组件频繁销毁重建？ |

---

## 维度 6：细节打磨

| 检查项 | 说明 |
|---|---|
| 链接类按钮 | 是否有 `target="_blank"` 的外链但缺少 `rel="noopener noreferrer"`？ |
| 图片 fallback | `<img>` / `<el-image>` 加载失败是否有占位 fallback？ |
| 长文本截断 | 长文本是否有截断 + tooltip 展示完整内容？ |
| 滚动位置记忆 | 页面切换/返回时是否有滚动位置丢失问题（需 `<keep-alive>` 或手动保存）？ |

---

## 维度 7：网络与性能

| 检查项 | 说明 |
|---|---|
| 并行请求 | 是否存在可并行发送的串行请求？改用 `Promise.all` / `Promise.allSettled` |
| 定时器清理 | 是否存在未清理的定时器/轮询（`setInterval`/`setTimeout` 未在 `onUnmounted` 清除）？ |
| 大列表优化 | 大列表是否有分页/虚拟滚动？ |

---
